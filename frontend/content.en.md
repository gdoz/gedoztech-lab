# ‚öôÔ∏è Operating Model

## Product Lines & Mini-Products

The Lab is structured into **Product Lines**, each containing multiple applications.  
Every application is treated as a **mini-product** with its own lifecycle, success criteria, stack, documentation, and governance.

## Process Method & Framework

The primary delivery system is **Kanban**, applied not as a board but as a **thinking model**:

- Visualize work  
- Limit work in progress  
- Manage flow  
- Improve collaboratively  
- Deliver continuously  

Scrum and other frameworks may be adopted for future product lines.

## Product Lifecycle

Every mini-product follows a consistent lifecycle:

### **Discovery**
Understand the problem, define scope and constraints, select stack and platform, and outline architecture, DevEx/UX, and initial technical direction.

### **Delivery**
Build minimum viable value, implement tests, establish security controls, set up CI/CD, automate, deploy, and measure flow.

### **Learning & Insights**
Analyze flow efficiency, cycle time, lead time, health signals, developer experience, maintainability, and (when relevant) cloud cost.

Continuous refinement drives continuous evolution.

# üìê Processes

> *"Kanban is not just a board ‚Äî it is the operating system of the Lab."*

### Kanban Implementation

GedozTech Lab uses a **dual-lane Kanban system**:

- **Discovery (Upstream)**  
- **Delivery (Downstream)**  

Both coexist in a unified flow pipeline.

### Core Policies

- Clear WIP limits  
- Definition of Ready (Initial Checklist)  
- Definition of Done (Closing Checklist)  
- Review expectations  
- Testing & security baselines  
- No new work starts if WIP is full  
- Kaizen log for process evolution  

### Core Kanban Artifacts

- Kanban Policy  
- Checklists (DoR/DoD)  
- Board Guidelines  
- Public Boards  
- Process Notes  
- Kaizen Log  
- Retrospectives  
- Metrics & Insights Dashboard  

### Idea ‚Üí Discovery ‚Üí Delivery Flow

1. Ideas are first registered in a public Google Sheets product index.  
2. Automated scripts generate tasks across the Lab‚Äôs management platforms.  
3. Discovery begins and flows naturally into Delivery.  
4. Delivery produces value and insights for continuous improvement.
